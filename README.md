# Design Principles for Scaling Enterprise Agents

Twelve principles for governing AI agents at enterprise scale — from defining outcomes to retirement.

## Files

- **design-principles-v2.md** — The principles (vendor-neutral)
- **applied-to-snowflake.md** — Mapping each principle to Snowflake features
- **adoption-playbook.md** — Playbook for driving organization-wide adoption of the governed path

## Quick reference

| Phase | Principles |
|---|---|
| Define it | P1 Outcomes, P2 Identity, P3 Front door |
| Build it | P4 Specs as code, P5 Reuse, P6 Vocabulary |
| Run it | P7 Least privilege, P8 Gates, P9 Trust |
| Keep it | P10 Observability, P11 Lifecycle, P12 Pave the road |

## How it fits together

The 12 Design Principles are a **platform framework** — they define how production agents are built, governed, observed, and retired. They prevent sprawl and make agents reliable and easy to consume. They also serve as a reference for companies to develop their own internal agent guidelines (e.g. a company skill that encodes your organization's standards).

The **Adoption Playbook** addresses the other half: how do you get your entire organization to actually use the governed path?

> The principles assume adoption. This playbook creates it.
